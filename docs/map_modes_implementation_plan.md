# План реализации карт: Categorical сначала, Continuous затем

## Контекст и решения

- Реализуем изменения **поэтапно**: сначала карты для кластеров (categorical), затем улучшения для непрерывных параметров (continuous).
- Интерактивные карты (web/plotly/leaflet) в этот план **не входят** — это отдельный крупный патч.
- Для continuous-режима сохраняем текущий принцип построения и оставляем colormap **`jet` по умолчанию** (как в текущем поведении).

---

## Scope

### In scope
1. Режимы карты в UI: `Continuous (Kriging)` и `Categorical (Cluster)`.
2. Отдельный рендер-пайплайн для категориальных меток.
3. Улучшения качества categorical-карт (маска области, uncertainty, сглаживание границ).
4. После стабилизации — улучшения continuous-карт (без смены default cmap).

### Out of scope
1. Интерактивная web-визуализация.
2. Глобальная переработка всей архитектуры визуализации за один релиз.

---

## Этап 1. UI-режимы и маршрутизация (фундамент)

### 1.1. Изменения формы карты (`draw_map_form.ui`)

Добавить логический переключатель режима:
- `comboBox_map_mode`:
  - `Continuous (Kriging)` (default)
  - `Categorical (Cluster)`

Добавить блоки контролов:
- Базовые (общие):
  - grid
  - sparse
  - cmap
  - show source points
  - show contours
- Категориальные:
  - method: `Nearest` / `Probability`
  - clip to data area
  - show uncertainty
  - boundary smoothing (Off/Low/Medium/High)
- Kriging (continuous):
  - estimator
  - var_model
  - dist_func
  - bin_func
  - nlags
  - smoothing/filter

#### Требования совместимости
- Сохранить objectName у существующих контролов, используемых кодом (например, `spinBox_grid`, `spinBox_sparse`, `comboBox_cmap`, `checkBox_filt`, `spinBox_filt`, `pushButton_map`).
- Новые контролы добавлять отдельными objectName.

### 1.2. Логика видимости контролов

- При `Continuous (Kriging)`:
  - Kriging controls активны
  - Categorical controls скрыты/disabled
- При `Categorical (Cluster)`:
  - Categorical controls активны
  - Kriging controls скрыты/disabled

### 1.3. Маршрутизация в коде отрисовки

В обработчике `DRAW`:
- `map_mode == Continuous` → текущий kriging-рендер.
- `map_mode == Categorical` → categorical-рендер.

### Definition of Done (Этап 1)
- UI переключает режимы без ошибок.
- Старый continuous-сценарий работает без регрессий.
- Кластерный сценарий запускает отдельный categorical-путь.

## Этап 2. Categorical v1 (стабильный production baseline)

### 2.1. Базовый алгоритм категориальной карты

- Вход: точки `(x, y)` и `cluster_label`.
- Сетка: регулярная, с контролируемым resolution.
- Метод v1: `Nearest` (устойчивый baseline).
- Визуализация:
  - полупрозрачная цветовая подложка классов,
  - точки исходных наблюдений,
  - границы/контуры классов,
  - отдельный стиль для noise (`label = -1`).

### 2.2. Маска области данных (clip)

- Реализовать обрезку результата интерполяции по области покрытия точками:
  - минимум: convex hull;
  - опционально: alpha-shape (если понадобится лучшее прилегание).
- При выключенном clip — поведение как сейчас (интерполяция на всю сетку).

### 2.3. Цвета и легенда для категорий

- Фиксированная таблица цветов кластеров (детерминированно по label).
- Отдельный цвет и подпись для noise.
- Легенда в формате `cluster N`.

### 2.4. Параметры производительности

- Диапазон resolution (с ограничениями min/max).
- Пресеты:
  - Fast (низкая сетка)
  - Balanced (средняя)
  - Detailed (высокая)

### Definition of Done (Этап 2)
- Для кластеров карта строится устойчиво и быстро.
- Нет «фантомной» заливки за пределами покрытия (при clip=ON).
- Визуализация кластеров читаема для пользователя.

## Этап 3. Categorical v2 (Probability + Uncertainty)

### 3.1. Метод Probability

Реализовать альтернативу `Nearest`:
- Для каждого класса вычислять score/probability field на сетке.
- Итоговый класс в ячейке: `argmax(prob_k)`.
- Опционально температурный параметр/степень сглаживания для контроля резкости границ.

### 3.2. Слой неопределенности

Добавить показатель uncertainty, например:
- `uncertainty = 1 - (p_top1 - p_top2)`
или аналогичный margin-based индекс.

Отображение uncertainty:
- прозрачность слоя,
- штриховка,
- отдельная полупрозрачная карта поверх категорий.

### 3.3. Сглаживание границ

- Постпроцессинг только для категориальной карты:
  - морфологические операции малого радиуса;
  - контроль уровня сглаживания (Off/Low/Medium/High).
- Ограничение: сглаживание не должно ломать локальную согласованность с исходными точками.

### Definition of Done (Этап 3)
- Доступны два метода (`Nearest`, `Probability`).
- Есть понятный слой uncertainty.
- Границы выглядят чище без потери адекватности.

## Этап 4. Continuous v1 (детальный план, с сохранением `jet` по умолчанию)

> Важно: на этом этапе **не меняем default colormap**. Для continuous остается `jet`.

### 4.1. Подготовка и инвентаризация

1. Зафиксировать текущий pipeline continuous-отрисовки:
   - вариограмма,
   - kriging,
   - optional filter,
   - contour + pcolormesh + scatter.
2. Собрать baseline-метрики:
   - время построения,
   - частота ошибок (`ValueError`, `LinAlgError`, `MemoryError`),
   - типовые размеры сетки и число точек.

### 4.2. Улучшение устойчивости расчета

1. Защитные проверки входа:
   - минимальное число точек,
   - проверка вырожденности координат,
   - предупреждение при плохом покрытии области.
2. Fallback-стратегия:
   - при ошибке kriging переключаться на более простой интерполятор (например, nearest/IDW fallback для визуализации),
   - выводить понятный `set_info` с причиной fallback.
3. Профилирование узких мест:
   - variogram fit,
   - transform на сетке,
   - постфильтрация.

### 4.3. Маска области для continuous

1. Добавить опцию `clip to data area` и для continuous.
2. Поведение:
   - ON: рисуем поле только внутри области покрытия,
   - OFF: текущее поведение полной сетки.
3. Сравнить визуально «до/после» на sparse-наборах.

### 4.4. Управление качеством и скоростью

1. Ввести пресеты качества continuous:
   - Fast / Balanced / Detailed.
2. Пресеты управляют:
   - grid resolution,
   - sparse sampling,
   - параметры фильтра.
3. Авто-рекомендация пресета:
   - по числу точек,
   - по площади участка,
   - по времени последнего построения.

### 4.5. Улучшение подписи и диагностики карты

1. В заголовок карты добавить компактный техпаспорт:
   - var_model,
   - estimator,
   - grid,
   - filter on/off,
   - clip on/off,
   - режим качества.
2. В лог/`set_info` писать:
   - длительность этапов,
   - fallback (если был),
   - предупреждения о качестве покрытия.

### 4.6. (Опционально) Слой доверия для continuous

- Если возможно получать оценку ошибки интерполяции — показывать uncertainty overlay.
- Если нет прямой sigma — использовать proxy по локальной плотности/дистанциям.

### Definition of Done (Этап 4)
- Continuous строится не хуже текущего baseline.
- `jet` остается default.
- Появляются clip/presets/диагностика/устойчивые fallback.
- Нет заметных регрессий производительности в типовых кейсах.

## Этап 5. Continuous v2 (тонкая оптимизация и UX)

### 5.1. Автоподбор параметров continuous

- Рекомендации по `grid`, `sparse`, `nlags`, фильтру.
- Rule-based сперва, затем (при желании) data-driven.

### 5.2. Кеширование промежуточных результатов

- Кешировать сетку, variogram fit или промежуточные transform (где возможно).
- Инвалидация кеша при смене параметров и входных данных.

### 5.3. Регрессионный контроль визуальных артефактов

- Набор эталонных кейсов (small/medium/large, sparse/dense).
- Скрипт сравнения (время + базовые статистики поля).

### Definition of Done (Этап 5)
- Continuous режим стабильно предсказуем по времени.
- Пользователь получает внятные presets и меньше ручных «подкруток».

## Тестирование (общий чеклист)

### Функциональные
- Переключение map mode и корректная видимость контролов.
- Построение карты для:
  - categorical nearest,
  - categorical probability,
  - continuous kriging.
- Корректная работа clip ON/OFF.

### Нагрузочные
- Малые, средние, большие наборы точек.
- Resolution на min/default/max.

### Негативные
- Пустые данные,
- слишком мало точек,
- вырожденные координаты,
- нестабильные параметры вариограммы.

### Регрессии
- Старые сценарии построения карт не ломаются.
- Default continuous colormap остается `jet`.
