# План разработки вкладки скважинного каротажа в модуле кластерного анализа

Документ фиксирует поэтапную реализацию вкладки скважинного каротажа на базе уже добавленной структуры БД и интерфейса в `model_cluster.py`.

## Цель этапа

Сделать рабочий MVP-процесс подготовки набора для кластеризации:
1. управление датасетами;
2. отбор скважин и интервалов каротажа;
3. добавление/удаление каротажных параметров;
4. сбор итоговых данных в `data`;
5. индикация состояния собранности набора и авто-сброс при изменениях.

---

## Этап 1. Управление наборами (dataset) и отображение в `combobox_cluster_well_set`

### 1.1. Добавление набора
- Источник имени набора: `lineEdit_string`.
- При нажатии на кнопку добавления:
  - валидировать непустое имя;
  - проверить отсутствие дубликата в рамках проекта/контекста;
  - создать запись набора в БД;
  - обновить модель UI;
  - выбрать новый набор в `combobox_cluster_well_set`.

### 1.2. Удаление набора
- Подтверждение удаления (диалог).
- Каскадное удаление/очистка зависимых сущностей набора:
  - выбранные скважины;
  - интервалы по скважинам;
  - выбранные каротажи;
  - собранные данные `data`.
- После удаления:
  - обновить `combobox_cluster_well_set`;
  - выбрать ближайший доступный набор или очистить форму.

### 1.3. Отображение наборов в combobox
- При открытии вкладки и при изменениях состава наборов делать перечитывание списка.
- Использовать стабильный `id` набора как значение элемента, отображаемое имя — пользовательское.
- При смене текущего набора загружать все связанные состояния на форме.

**Критерий готовности этапа:** можно создать/удалить наборы, и они корректно синхронизированы с `combobox_cluster_well_set`.

---

## Этап 2. Добавление и удаление скважин в наборе

### 2.1. ADD WELL (добавление одной скважины)
- По нажатию `ADD WELL` брать текущую выбранную скважину из `listWidget_well`.
- Перед добавлением проверять:
  - что скважина выбрана;
  - что у скважины есть каротаж (`well_log` не пуст);
  - что скважина еще не добавлена в текущий набор.
- После успешного добавления обновлять список скважин набора и статус собранности.

### 2.2. ADD WELLS (массовое добавление по радиусу от профилей текущего объекта)
- По нажатию `ADD WELLS`:
  - прочитать радиус из `spinBox_well_distance`;
  - получить скважины по расстоянию от **профилей текущего объекта** (использовать существующую функцию, аналогичную логике, которая применяется при построении карты при включенном режиме показа скважин);
  - отфильтровать только скважины, у которых есть каротаж (`well_log` не пуст);
  - добавить их в набор без дублей.

### 2.3. Ручное удаление скважин
- Удаление выделенных скважин из списка скважин набора.
- При удалении скважины очищать связанные с ней:
  - выбранные интервалы;
  - привязанные каротажи по этой скважине;
  - строки `data`, относящиеся к этой скважине.

### 2.4. Выбор интервала по двойному клику
- Двойной клик по скважине в списке открывает `form_well_log`.
- В `form_well_log` добавить отдельную кнопку «Принять интервал для кластерного набора».
- По кнопке сохранять выбранный интервал в сущность текущего набора + скважины.
- В списке скважин показывать маркер, что интервал выбран (иконка/цвет/текст).

**Критерий готовности этапа:** пользователь может добавить скважины по одной (`ADD WELL`) и массово (`ADD WELLS`), удалить их, и для каждой выбрать интервал через `form_well_log`.

---

## Этап 3. Работа с каротажными параметрами (без калькулятора)

### 3.1. Добавление одного параметра
- Для выделенной скважины запускать `form_well_log`.
- Через отдельную кнопку в форме добавлять выбранную кривую в набор.
- Имя параметра сохранять в **каноническом** виде из соответствующей таблицы канонических имен.
- Если каноническое имя не найдено:
  - показать информативное сообщение пользователю;
  - пропустить добавление этой кривой.

> Примечание: создание/редактирование канонических имен и алиасов не входит в этот план (будет отдельный документ).

### 3.2. ADD ALL (массовое добавление параметров)
- Добавить все доступные «чистые» каротажи без вычисляемых формул.
- В списке параметров показывать статистику покрытия: на скольких скважинах набора есть данный каротаж.
- Формат отображения, например: `GR (12/18)`.

### 3.3. Удаление параметров
- Удаление выбранного параметра из набора.
- При удалении параметра удалять связанные колонки/значения из `data` (или помечать набор как несобранный с последующей пересборкой).

**Критерий готовности этапа:** параметры добавляются по одному и массово, отображается покрытие по скважинам, удаление работает корректно.

---

## Этап 4. COLLECT: сбор данных в `data`

### 4.1. Кнопка COLLECT
- Собрать данные по всем выбранным:
  - скважинам;
  - интервалам скважин;
  - каротажным параметрам.
- Сохранять результат в `data`, привязанный к текущему dataset.

### 4.2. Проверки перед сборкой
- Нет скважин -> блокирующее сообщение.
- Не задан интервал хотя бы для части скважин -> предупреждение с выбором политики (пропустить/остановить).
- Нет выбранных параметров -> блокирующее сообщение.

### 4.3. Индикация состояния собранности
- Цвет кнопки `COLLECT` отражает состояние:
  - **не собран** (например, серый/красный);
  - **собран и актуален** (например, зеленый).

### 4.4. Автосброс собранного набора при изменениях
При любом изменении конфигурации набора удалять (или инвалидировать) текущий `data` и переводить набор в состояние «не собран»:
- добавление/удаление скважины;
- изменение интервала скважины;
- добавление/удаление параметра;
- изменение других параметров, влияющих на состав выборки.

**Критерий готовности этапа:** COLLECT формирует `data`, а любое изменение входных параметров автоматически сбрасывает «собранность».

## Этап 5. UX-доработки и контроль качества

### 5.1. UX
- Единые сообщения об ошибках/предупреждениях.
- Подсказки в UI (tooltip) для кнопок `ADD WELLS`, `ADD ALL`, `COLLECT`.
- Отображение текущего статуса набора (скважины/параметры/собранность).

### 5.2. Минимальный набор тест-кейсов
1. Создание/удаление набора и синхронизация с combobox.
2. ADD WELL: добавление выбранной скважины из `listWidget_well` с проверками выбора/дублей/наличия каротажа.
3. ADD WELLS: добавление по радиусу от профилей текущего объекта и фильтрация по наличию каротажа.
4. Выбор интервала через двойной клик + `form_well_log`.
5. Добавление одного параметра с успешной и неуспешной канонизацией.
6. ADD ALL и корректность статистики покрытия.
7. Удаление скважин/параметров и очистка зависимых данных.
8. COLLECT и смена цвета кнопки.
9. Автосброс собранного `data` после любого изменения входных настроек.

## Порядок реализации (рекомендуемый)

1. Этап 1 (dataset + combobox).
2. Этап 2 (скважины + интервалы).
3. Этап 3 (параметры без калькулятора).
4. Этап 4 (COLLECT + состояние собранности).
5. Этап 5 (UX и тест-кейсы).

Такой порядок минимизирует переделки: сначала фиксируется контейнер набора (dataset), затем его состав, и только после этого — сборка итогового `data`.
