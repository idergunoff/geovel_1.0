# План реализации: ограничение минимального размера кластера через спинбокс (в образцах)

## 1) Цель изменения

Снизить вероятность выбора «ложно лучших» параметров кластеризации, при которых возникают слишком маленькие кластеры (вплоть до 1 образца), за счёт добавления пользовательского ограничения:

- **спинбокс `Минимальный размер кластера`** в **образцах**;
- **кнопка сброса** значения к рекомендованному дефолту = **5% от текущей выборки**;
- **динамический максимум спинбокса** = **50% от размера текущей выборки**.

---

## 2) Функциональные требования

### 2.1 Новые элементы интерфейса

1. Спинбокс `Минимальный размер кластера (образцы)`.
2. Кнопка `Сброс к 5%` (или эквивалентная подпись).
3. Подсказка/tooltip с пояснением:
   - ограничение применяется в автоподборе параметров;
   - кластеры меньшего размера считаются невалидными.

### 2.2 Правила значений спинбокса

Для текущей выборки размера `N`:

- `min_value = 1` (или 2/3 по бизнес-решению);
- `max_value = max(min_value, floor(0.5 * N))`;
- `default_5pct = max(min_value, ceil(0.05 * N))`.

> Рекомендуется сохранить текущее решение команды по нижней границе (`1`, `2` или `3`) и использовать его единообразно в UI и валидации расчёта.

### 2.3 Поведение при смене объекта исследований

При загрузке/выборе нового объекта (и новом `N`):

1. Пересчитать `max_value` и `default_5pct`.
2. Обновить диапазон спинбокса (`min/max`).
3. Установить значение по умолчанию `default_5pct` **только если**:
   - это первый вход в модуль, или
   - пользователь нажал «Сброс к 5%», или
   - выбранное ранее значение вышло за новый диапазон.
4. Если сохранённое значение больше нового `max_value`, выполнить clamping до `max_value` и показать короткое уведомление.

### 2.4 Кнопка «Сброс к 5%»

По нажатию:

1. Взять текущее `N`.
2. Пересчитать `default_5pct = max(min_value, ceil(0.05 * N))`.
3. Установить это значение в спинбокс.
4. Обновить внутреннее состояние параметров автоподбора.

---

## 3) Изменения в логике автоподбора кластеризации

### 3.1 Применение ограничения

Для каждого кандидата параметров:

1. Построить разбиение на кластеры.
2. Посчитать размеры всех кластеров (исключая шум, если алгоритм его поддерживает).
3. Если существует кластер размера `< min_cluster_samples`, кандидат пометить как **невалидный**.
4. Среди валидных кандидатов выполнить ранжирование по основной метрике качества.

### 3.2 Особые случаи

1. **Нет валидных кандидатов**:
   - вернуть понятное сообщение пользователю;
   - предложить снизить порог или нажать «Сброс к 5%».
2. **Алгоритмы с шумом (DBSCAN/HDBSCAN)**:
   - шум не считать кластером;
   - ограничение применять только к реальным кластерам.
3. **Малые выборки** (`N < 20` и т.п.):
   - проверить, что `max_value` не ломает UX (например, при `N=7` максимум будет 3).

---

## 4) Состояние и сохранение настроек

1. Хранить выбранное пользователем значение как часть состояния модуля кластеризации.
2. Разделить:
   - `current_user_value` (текущее выбранное);
   - `recommended_default_value` (вычисленное 5%).
3. Не перезаписывать `current_user_value` автоматически без явной причины (см. п. 2.3).

---

## 5) Валидация и обработка ошибок

1. На уровне UI не позволять выйти за диапазон `[min_value, max_value]`.
2. На уровне backend/сервиса автоподбора повторно валидировать входной параметр.
3. В логах фиксировать:
   - `N`, `min_cluster_samples`, число невалидных кандидатов,
   - причину отбраковки (кластеры меньше порога).

---

## 6) План внедрения по этапам

### Этап 1. Подготовка модели параметров

- Добавить в DTO/конфиг параметр `min_cluster_samples`.
- Добавить вычисление:
  - `recommended_default_value = ceil(0.05 * N)`;
  - `max_spinbox_value = floor(0.5 * N)`.

### Этап 2. Обновление UI

- Добавить спинбокс (в образцах).
- Добавить кнопку `Сброс к 5%`.
- Привязать обработчики:
  - `onObjectChanged` → обновление диапазона и дефолтов;
  - `onResetTo5Percent` → установка рекомендованного значения.

### Этап 3. Интеграция с автоподбором

- Перед расчетом метрик добавить фильтрацию кандидатов по `min_cluster_samples`.
- Отразить статус кандидатов (валидный/невалидный).
- Обработать сценарий отсутствия валидных вариантов.

### Этап 4. Логирование и диагностика

- Добавить структурированные логи отбраковки кандидатов.
- Добавить краткую сводку в результатах подбора (сколько кандидатов отброшено).

### Этап 5. UX-полировка

- Tooltip/подсказка.
- Сообщения об автокоррекции значения при смене объекта.
- Единый текст ошибок/уведомлений.

## 7) Тест-план

### 7.1 Unit-тесты

1. Расчёт дефолта:
   - `N=100` → `5`;
   - `N=101` → `6` (ceil);
   - малые `N` с учетом нижней границы.
2. Расчёт максимума:
   - `N=100` → `50`;
   - `N=9` → `4`.
3. Clamping значения при смене `N`.

### 7.2 Интеграционные тесты

1. Смена объекта:
   - диапазон обновляется;
   - кнопка «Сброс к 5%» выставляет корректное значение.
2. Автоподбор:
   - кандидаты с малыми кластерами отбрасываются;
   - при отсутствии валидных кандидатов возвращается корректное сообщение.

### 7.3 UI/ручные проверки

1. Спинбокс не выходит за `max = 50% от N`.
2. При `N` в разных диапазонах (малый/средний/большой) поведение ожидаемое.
3. Отображение уведомлений и подсказок читаемое и однозначное.

---

## 8) Критерии готовности (Definition of Done)

1. Спинбокс работает в образцах, с динамическим максимумом `50%` от `N`.
2. Кнопка «Сброс к 5%» устанавливает `ceil(0.05 * N)`.
3. Автоподбор не выбирает кандидаты с кластерами ниже порога.
4. Есть обработка сценария «нет валидных кандидатов».
5. Покрытие ключевой логики unit/integration-тестами.
6. Обновлена документация/подсказки в интерфейсе.

---

## 9) Опциональные улучшения (после базового релиза)

1. Переключатель режима:
   - **Строгий**: отбрасывать кандидаты;
   - **Мягкий**: штрафовать в score.
2. Показ в отчете подбора:
   - распределение размеров кластеров для топ-N кандидатов.
3. Отдельный дефолт по типам данных/проектов (не только 5%).
